    <h2>Description</h2>
    <p>The <code class="element">button</code> element, having the "submit" value in its <code class="attribute">type</code> attribute, represents a button that, when pressed, submits the <a href="form.html"><code class="element">form</code></a> it belongs to. The label of a <code class="element">button</code> is represented by the content of the element.</p>
    <p>With the arrival of <abbr title="Hypertext Markup Language">HTML</abbr>5, several new attributes have been added to this element (<code class="attribute">formaction</code>, <code class="attribute">formenctype</code>, <code class="attribute">formmethod</code>, <code class="attribute">formnovalidate</code> and <code class="attribute">formtarget</code>) that define and override certain parameters pertaining how the <a href="form.html"><code class="element">form</code></a> must be submitted. These new attributes can be used, for example, to provide more than one submit button in one <a href="form.html"><code class="element">form</code></a>, and make each of them perform a different type of submission.</p>
    <p class="info message-right"><span class="image"></span>In contrast with the <a href="input.html"><code class="element">input</code> (type=submit) element</a>, this type of button can contain other non-interactive elements.</p>

    <section id="specific-attributes">
      <h3>Specific attributes</h3>
      <details class="attribute">
        <summary>
          <h4>autofocus</h4>
        </summary>
        <p>A boolean value instructing the browser to set the focus to this control when the document has finished loading or when the <a href="dialog.html"><code class="element">dialog</code></a> where the control finds itself is shown. If the attribute has the value "autofocus" or the empty string (""), or if it's just present, the control should get the focus as soon as possible, after the page or <a href="dialog.html"><code class="element">dialog</code></a> has been loaded.</p>
        <p class="code-view-title"><span>Example</span></p>

<pre class="code-view"><code><span class="line"><span class="number" data-num="1."></span><span class="line-1"><span class="tag">&lt;button<span class="atn"> type=</span><span class="atv">&quot;submit&quot;</span><span class="atn"> autofocus&gt;</span></span>Submit form<span class="tag">&lt;/button&gt;</span></span></span>
</code></pre>

      </details>
      <details class="attribute">
        <summary>
          <h4>disabled</h4>
        </summary>
        <p>A boolean value indicating wether the control is disabled or not. If the attribute takes the value "disabled" or the empty string (""), or if it's just present, the control will be disabled.</p>
        <p class="info"><span class="image"></span>Disabled controls are rendered greyed out (if visible), are blocked from user interaction and, more importantly, their values (if any) aren't sent when the <a href="form.html"><code class="element">form</code></a> is submitted.</p>
        <p class="code-view-title"><span>Example</span></p>

          <h4>formaction</h4>
        </summary>
        <p>A <a href="../uri-url.html"><abbr title="Uniform Resource Identifier">URI</abbr></a> indicating the location of the script responsible for the manipulation of the data sent by the <a href="form.html"><code class="element">form</code></a>.       <details class="attribute">
        <summary>
          <h4>formenctype</h4>
        </summary>
        <p>A value indicating the encoding method to be used for data when the form is submitted. There are three possible case-insensitive values:</p>
        <ul>
          <li><b>application/x-www-form-urlencoded</b>: spaces are replaced with plus signs ("+") and special characters are converted to <i>HEX</i> values. This is the default value.</li>
          <li><b>multipart/form-data</b>: no encoding is performed. This value is necessary for file uploads.</li>
          <li><b>text/plain</b>: only spaces are replaced by plus signs ("+").</li>
        </ul>
        <p class="info"><span class="image"></span>Remember you must use the value "multipart/form-data" whenever a file is going to be uploaded in the form. Without this configuration, file uploads will fail.</p>
        <p class="info"><span class="image"></span>This attribute is new in <abbr title="Hypertext Markup Language">HTML</abbr>5 and, if present, overrides the <code class="attribute">action</code> attribute of the <a href="form.html"><code class="element">form</code> element</a>. This allows authors to provide one <a href="form.html"><code class="element">form</code></a> with two or more buttons that perform different types of submissions.</p>
        <p class="code-view-title"><span>Example</span></p>

<pre class="code-view"><code><span class="line"><span class="number" data-num="1."></span><span class="line-1"><span class="tag">&lt;form<span class="atn"> action=</span><span class="atv">&quot;../../form-result.php&quot;</span><span class="atn"> method=</span><span class="atv">&quot;post&quot;</span><span class="atn"> target=</span><span class="atv">&quot;_blank&quot;</span>&gt;</span></span></span>

      <details class="attribute">
        <summary>
          <h4>formmethod</h4>
        </summary>
        <p>The method browsers should use to send the form's data. There are three possible case-insensitive values:</p>
        <ul>
          <li><b>get</b>: data is attached to the <a href="../uri-url.html"><abbr title="Uniform Resource Locator">URL</abbr></a> of the request (the one provided in the <code class="attribute">action</code> attribute). The name-value pairs are arranged in the form "name=value" and separated each other with an ampersand sign ("&amp;"). All this string is appended to the request <a href="../uri-url.html"><abbr title="Uniform Resource Locator">URL</abbr></a> preceded by a question mark ("?"). For example, a GET string could be: "form-result.php?user=john&amp;pass=123456"</li>
          <li><b>post</b>: data is attached to the body of the request.</li>
          <li><b>dialog</b>: specific for forms inside a <a href="dialog.html"><code class="element">dialog</code> element</a>. Instructs the browser to close the dialog box upon form submission. Form results should be handled by scripts.</li>
        </ul>
        <p class="info"><span class="image"></span>This attribute is new in <abbr title="Hypertext Markup Language">HTML</abbr>5 and, if present, overrides the <code class="attribute">action</code> attribute of the <a href="form.html"><code class="element">form</code> element</a>. This allows authors to provide one <a href="form.html"><code class="element">form</code></a> with two or more buttons that perform different types of submissions.</p>
        <p class="code-view-title"><span>Example</span></p>

<pre class="code-view"><code><span class="line"><span class="number" data-num="1."></span><span class="line-1"><span class="tag">&lt;form<span class="atn"> action=</span><span class="atv">&quot;../../form-result.php&quot;</span><span class="atn"> target=</span><span class="atv">&quot;_blank&quot;</span>&gt;</span></span></span>

        <summary>
          <h4>formtarget</h4>
        </summary>
        <p>A value specifiyng where the results of the script in charge of processing the data should be loaded. This value can be a browsing-context name (like the value of the <code class="attribute">name</code> attribute of an <code class="element">iframe</code>) or any of the following values (case-insensitive):</p>
        <ul>
          <li><b>_blank</b>: in a new window/tab.</li>
          <li><b>_parent</b>: in the immediate parent context.</li>
          <li><b>_self</b>: in the same context that's containing the form.</li>
          <li><b>_top</b>: in the topmost context (the greatest parent context containing the form).</li>
        </ul>
        <p class="info"><span class="image"></span>This attribute is new in <abbr title="Hypertext Markup Language">HTML</abbr>5 and, if present, overrides the <code class="attribute">action</code> attribute of the <a href="form.html"><code class="element">form</code> element</a>. This allows authors to provide one <a href="form.html"><code class="element">form</code></a> with two or more buttons that perform different types of submissions.</p>

This allows authors to provide one <a href="form.html"><code class="element">form</code></a> with two or more buttons that perform different types of submissions.</p>
        <p class="info"><span class="image"></span>This attribute is part of the Constraint Validation <abbr title="Application Programming Interface">API</abbr>, a new feature that allows authors to provide <a href="form.html"><code class="element">form</code></a> validation with minimal programming intervention.</p>

        <p>A name for the control. This name will be sent by the browser to the processing agent, paired with the content of the <code class="attribute">value</code> attribute. Both attributes together will conform a name-value pair that will be used to process the form data.</p>

      <details class="attribute">
        <summary>
          <h4>type</h4>
        </summary>

        <p>A value indicating the expected behavior of the button. There are four possible values (case-insensitive) that will decide the default action carried out by the button when it's pressed:</p>
        <ul>
          <li><a href="button-button.html">button</a>: there's no default action associated to the control. The behavior of this type of buttons is usually provided by a script.</li>
          <li><a href="button-reset.html">reset</a>: the controls of the associated form are reset to their initial values.</li>
          <li><a href="button-submit.html">submit</a>: the associated form is submitted. This is the deafult value.</li>
          <li><a href="button-menu.html">menu</a>: the context menu associated to this button is deployed.</li>
        </ul>
        <p>When this attribute isn't present, the button behaves as a "submit" button.</p>

        <p class="code-view-title"><span>Example</span></p>
